## 核心新功能详细设计

### ★ 三个终极命题向导 (W1)

**交互流程**：
```
Step 1 → "这个世界是什么？" (描述世界本质 + 力量来源)
  ↓ AI 实时提示和扩展建议
Step 2 → "从何而来？" (历史起源 + 关键事件)
  ↓ AI 基于 Step 1 给出历史走向建议
Step 3 → "往何处去？" (命运走向 + 核心矛盾)
  ↓ AI 综合前两步给出冲突张力分析
Step 4 → 基础参数 (类型/基调/角色数/章节数)
Step 5 → 确认 → 开始创世
```

**AI 辅助**：每一步用户输入后，AI 实时给出：
- 世界观自洽性分析
- 可能的冲突点建议
- 参考世界观对标（"类似XXX的设定"）

### ★ 多视角故事采集 (S3)

**实现方案**：

1. **视角筛选**：选定角色后，筛选该角色参与的所有场景
2. **记忆重组**：从该角色的 episodic memory 重建叙事线
3. **信息屏蔽**：该角色不知道的信息不出现（其他角色的内心独白）
4. **情感着色**：基于该角色当时的情感状态调整叙述语气
5. **Writer Agent 重写**：以选定视角重新渲染每个场景

```python
# 伪代码
async def harvest_story(world_id, perspective_character_id, style):
    """从运行完的世界中采集一个角色视角的故事"""
    # 1. 获取该角色参与的所有场景
    scenes = get_scenes_involving(perspective_character_id)

    # 2. 为每个场景构建该角色的视角上下文
    for scene in scenes:
        char_memory = get_memory_at(perspective_character_id, scene.chapter, scene.index)
        char_emotion = get_emotion_at(perspective_character_id, scene.chapter, scene.index)

        # 3. 只包含该角色能感知到的信息
        visible_actions = filter_visible(scene.actions, perspective_character_id)

        # 4. Writer 以该视角重新叙述
        narrative = await writer.write_scene_perspective(
            actions=visible_actions,
            pov_character=perspective_character_id,
            pov_memory=char_memory,
            pov_emotion=char_emotion,
            style=style  # 第一人称/第三人称
        )
```
