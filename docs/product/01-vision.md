# 01 — 产品愿景

## 我们在做什么

**WorldEngine** 不是一个"AI 写作工具"。它是一个**世界模拟引擎**。

用户不是在"让 AI 写小说"，而是在：
1. **创造一个世界** — 物理法则、力量体系、文明版图、历史长河
2. **赋予居民灵魂** — 每个角色是独立 Agent，有自己的记忆、情感、目标、价值观
3. **按下"运行"** — 世界按照自身规则和时间线推进，角色自主行动、结盟、背叛、成长
4. **派出一个"史官"** — 从这个运行着的世界里，以任意视角，书写一段故事

这和传统 AI 小说的区别是：
- **传统**：输入提纲 → AI 填充文字 → 输出小说（**一次性消耗品**）
- **我们**：构建世界 → 世界运行 → 随时采集故事（**可持续资产**）

---

## 终极命题

每一个世界观的构建，最终都在回答三个问题：

### 🔵 我是谁
世界的本质是什么？力量的来源是什么？
角色在这个世界中的身份认同。
> "在一个灵气复苏的世界里，我是一个失去记忆的剑修"

### 🟢 我从哪里来
世界的历史。文明的起源。角色的过去。因果链的起点。
> "千年前的封印之战，我的前世是那个背叛了同门的人"

### 🔴 我要到哪里去
世界的命运走向。角色的终极选择。主题的最终表达。
> "当封印再次破碎，我选择牺牲自己完成前世未竟的赎罪"

每个世界必须在创建时回答这三个问题。它们是世界的**锚点**，所有角色的命运轨迹都围绕它们展开。

## 核心差异化

### 1. 角色不是"设定"，是"活着的 Agent"

| 传统 AI 小说 | WorldEngine |
|-------------|-------------|
| 角色是一段设定文本 | 角色是独立运行的 Agent |
| AI 控制角色说什么做什么 | Agent 根据性格、记忆、情感**自主决策** |
| 角色行为由大纲驱动 | 角色行为由内在动机驱动 |
| 换个视角要重新生成 | 同一个世界运行结果，可以从任意视角叙述 |

每个角色 Agent 拥有：
- **`agent.md`** — 身份档案（名字、背景、性格五维度、说话方式）
- **`soul.md`** — 灵魂核心（价值观、底线、内心冲突、命运母题）— **用户可编辑**
- **`memory/`** — 记忆系统（情景记忆、语义记忆、情感状态、关系网络）

### 2. 世界不是"背景板"，是"运行中的系统"

世界有自己的：
- **物理法则** — 力量体系的规则与限制
- **社会结构** — 势力、阵营、利益关系
- **时间线** — 历史事件的因果链
- **命运之手（God Agent）** — 在章与章之间做出命运级决策

### 3. 一个世界 → 无数故事

世界运行一次，可以：
- 以**主角 A 的视角**写一部成长史诗
- 以**反派 B 的视角**写一部悲剧
- 以**旁观者 C 的视角**写一部悬疑
- 以**God 的视角**写一部编年史
- 把**某个事件**单独展开成一个短篇

这就是为什么用户愿意花时间（和金钱）构建一个世界——它是**可复用的创作资产**。

### 4. "史官"，不是"作者"

最终输出小说的 Agent（Writer）不是在"创作"，而是在"记录"。
它基于世界已经发生的事件、角色的真实行动、情感的真实变化来叙述。
这保证了：
- **剧情连贯性** — 因为事件确实发生过
- **角色一致性** — 因为角色确实是那样想的
- **细节丰富度** — 因为世界确实有那些规则

## 目标用户

### 一级用户：网文/轻小说创作者
- 痛点：一个人构思世界观太累，写到后面角色崩坏
- 价值：世界观自动构建 + 角色一致性保障 + 多视角产出
- 场景：一个世界连载多部作品

### 二级用户：TRPG / 世界观爱好者
- 痛点：有丰富的世界设定但没时间/能力写成故事
- 价值：输入设定 → 看角色在自己的世界里活起来
- 场景：把自己的 TRPG 设定变成小说

### 三级用户：游戏/影视编剧团队
- 痛点：多人协作世界观容易不一致
- 价值：单一可信来源（Single Source of Truth）+ 自动推演
- 场景：原创 IP 世界观开发 + 多线叙事验证

### 四级用户：AI 创作探索者
- 痛点：对"AI 写的东西"不满意，觉得太平太假
- 价值：理解并参与 AI 创作过程，通过编辑 `soul.md` 影响角色命运
- 场景：与 AI 共创，享受"造物主"体验
